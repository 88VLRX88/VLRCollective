# VLRCollective

Проект где множество Агентов работают над Одним вопросом. (В связке с API lmstudio)

## Зачем это

По моим подсчетам параметры в 4 мета итерации при 3 агентах улучшают структуру кода и эффективность приблизительно на 20%. Позволяет давать более полные и многосторонние ответы на вопросы и продумывать узкие моменты в сложной архитектуре(Агенты никогда не перестают накидывать идеи).

## Как запустить

Скопируй репозиторий:

```bash
git clone https://github.com/твой-логин/VLRCollective.git
cd VLRCollective
```

Установи зависимости:

```bash
pip install -r requirements.txt
```

Запусти бэкенд:

```bash
python backend/server.py
```

Открой браузер и зайди на `http://localhost:5000`. Введи любой ID сессии и задай вопрос.

ВАЖНО: в папке collective должна находится папка одноименная с ID сессии для успешного запуска проекта.

## Структура

```
├── collective/        # папка с логами — каждая сессия в своей папке 
├── backend/           # сервер на FastAPI + VLRCollective.py
├── css/               # стили
├── js/                # фронтенд
└── index.html         # одна страница — всё нужное
```

## Что можно настроить

В веб-интерфейсе:
- добавить или удалить агента
- поменять его характер
- задать количество итераций (сколько раз каждый агент напишет свое сообщение)

## Как это выглядит

Ты вводишь вопрос, агенты начинают дискуссию, синтезатор собирает итог. Логи и синтез видны в реальном времени.

## Почему это может быть полезно

- Хочешь услышать разные точки зрения на одну задачу
- Нужно, чтобы ИИ не просто дал ответ, а показал, как к нему пришли
- Надоело читать однотипные ответы — агенты спорят, значит, и результат интереснее
- Код и прочие задачи требующие продумывания архитектуры выполняются эффективнее.
